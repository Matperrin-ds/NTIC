Reinforcement Learning (RL)



Reinforcement learning is a sub-field of Machine Learning in which algorithms learn through experience. Just like living animals, they try several solutions in order to solve a problem. Sometimes they try and improve a strategy they already know works well, and sometimes they choose to explore new ways instead. This tradeoff is called "Exploitation" vs "Exploration".


In order so solve a problem with Reinforcement Learning algorithms, it has to be addressed correctly. Every RL problem is made of 2 parts : the environment where actions take place (e.g coliseum), and the agent, who takes those actions (e.g gladiator).


For example, say you want to teach a robotic arm how to play darts. The environment is the software simulating the target, some physical laws, and the mathematical model for the arm. It holds the informations about what happens to the dart when a certain command is given to the arm. The environment is also given a "Reward function" that rates a shot : the score gets greater the closer the dart is to the center of the target.


On the other hand, the agent (e.g the chip controlling the arm) has no knowledge of the environment and its physical laws. At each moment in time, it only knows the available actions and can observe the reward it gets from taking them. Its goal is to find the strategy, or more precisely the "policy" that maximizes the obtained reward.


Sometimes, single-agent Reinforcement Learning isn't enough to fully model cooperative behaviours. For example, when modelling crowds that walk through each other, researchers are planning to use several agents to increase the realism of their simulations. Several examples show that in this case, Multi-Agent Reinforcement Learning (MARL) can provide interesting solutions.


The idea is simple : instead of having only one agent making decisions, there are several of them. Every time step each agent takes an action, then one gets a list of actions corresponding to the actions taken by each agent. Finally, the environment evolves from one state to another with respect to this list of actions, instead of a single one.
